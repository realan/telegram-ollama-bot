# Phase Reports

В этой папке хранятся краткие отчеты по завершенным фазам.

Зачем это нужно:

- чтобы важные результаты не терялись в чате
- чтобы фиксировать, что реально было сделано
- чтобы помнить отклонения от spec
- чтобы передавать следующей фазе краткий итог без чтения всей переписки

## Формат

Рекомендуемый файл на фазу:

- `_docs/reports/phase-1-design-report.md`
- `_docs/reports/phase-2-memory-report.md`

Шаблон:

- [phase-report.template.md](/Users/user/projects/hw1/_docs/reports/phase-report.template.md)

## Как использовать

После завершения фазы:

1. Создать отчет по шаблону.
2. Кратко зафиксировать:
   - что сделано
   - что не сделано
   - какие решения приняты
   - какие были отклонения от spec
   - что важно для следующей фазы
3. Обновить:
   - `_docs/roadmap.yaml`
   - `_docs/context-dump.md`
   - при необходимости `decisions/`

## Разделение ответственности

- `reports/` — краткая память о выполнении фазы
- `decisions/` — долгоживущие решения
- `context-dump.md` — актуальное текущее состояние проекта

Отчет не должен заменять ADR или context dump. Он нужен как короткий operational summary.
